## 2024-03-24 - Debouncing Continuous UI Events Linked to Heavy Canvas Rendering
**Learning:** Continuous input events on UI controls (like `<input type="color">` and `<input type="range">`) can trigger up to 60 events per second while a user interacts with them. If these input events directly invoke expensive operations, such as recompiling and redrawing an entire canvas with a large library (like `QRCodeStyling`), it immediately blocks the main thread and causes severe UI lag and jank. The application's input response degrades completely because the event loop is choked with render tasks.
**Action:** Always wrap heavy synchronous rendering operations behind a debounce function (e.g., `triggerUpdateDebounced()`) when they are bound to continuous inputs like `input`, `mousemove`, or `scroll`. This ensures the expensive operation only fires when the user pauses or stops their interaction, keeping the UI thread fluid.
